 28  ''' 
 29  Frycook depends on recipes and cookbooks to define how systems are built. 
 30  Recipes and cookbooks in turn use settings, environments, and packages to do 
 31  their work.  The settings and environments are passed around as dictionaries and 
 32  refer to the packages, which live on the disk as directories and files. 
 33   
 34  settings dictionary 
 35  =================== 
 36   
 37  There are a few settings that frycook depends on.  They're contained in a 
 38  dictionary called settings that's passed to the constructors for Cookbooks and 
 39  Recipes.  It has the following keys: 
 40   
 41  I{package_dir}: root of the packages hierarchy for the file copying routines to 
 42  look in 
 43   
 44  I{module_dir}: temporary directory for holding compiled mako modules during 
 45  template handling 
 46   
 47  I{file_ignores}: file patterns to ignore while copying package files 
 48   
 49  For any key containing 'dir' or 'path', if you include a tilde (~) in the value, 
 50  it will be replaced with the home directory of the user running frycooker, just 
 51  like in bash.  For this example, that would be the "package_dir" and 
 52  "module_dir" keys. 
 53   
 54  example:: 
 55   
 56    { 
 57    "package_dir": "~/bigawesomecorp/admin/packages/", 
 58    "module_dir": "/tmp/bigawesomecorp/mako_modules", 
 59    "tmp_dir": "/tmp", 
 60    "file_ignores": ".*~" 
 61    } 
 62   
 63  environment dictionary 
 64  ====================== 
 65   
 66  This contains all the metadata about the computers that the recipes and 
 67  cookbooks will be applied to.  Most of its data is directly relevant to 
 68  specific recipes and is filled in depending on the recipes' needs.  It's a 
 69  dictionary with three main sections that should always be there: 
 70   
 71  I{users}: a list of the users that recipes could reference, with such things as 
 72  public ssh keys and email addresses for them 
 73   
 74  I{computers}: a list of computers in the system, with such things as ip 
 75  addresses 
 76   
 77  I{groups}: groups of computers that will be addressed as a unit 
 78   
 79  Just like for the settings dictionary, any key containing 'dir' or 'path' and 
 80  including a tilde (~) in the value will have the tilde replaced with the home 
 81  directory of the user running frycooker.

114  packages directory 
115  ================== 
116   
117  This contains all the files needed by the recipes.  There is one sub-directory 
118  per package, and each package generally corresponds to a recipe.  Within each 
119  package the directories and files are laid out the exact same as they will be 
120  on the target systems.  Any files with a .tmplt extension will be processed as 
121  mako templates before being copied out to computers.  The fck_metadata.txt 
122  files define the ownership and permissions for the files and directories when 
123  they're copied to the target system.  Here's an example layout:: 
124   
125      packages                 # directory for the package files 
126        hosts                  # root for hosts package files 
127          etc                  # corresponds to /etc on the target server 
128            hosts.tmplt        # template that becomes /etc/hosts on the target 
129                               # server 
130        nginx                  # root for nginx package files 
131          etc                  # corresponds to /etc directory on the target server 
132            default            # corresponds to /etc/default directory on the 
133                               # target server 
134              nginx            # corresponds to /etc/default/nginx file on the 
135                               # target server 
136            nginx              # corresponds to /etc/nginx directory on target 
137                               # server 
138              fck_metadata.txt # define ownership and perms for the /etc/nginx 
139                               # directory on the server 
140              nginx.conf       # corresponds to /etc/nginx/nginx.conf file on 
141                               # target server 
142              conf.d           # corresponds to /etc/nginx/conf.d directory on 
143                               # target server 
144              sites-available  # corresponds to /etc/nginx/sites-available 
145                               # directory on target server 
146                default        # corresponds to /etc/nginx/sites-available/default 
147                               # directory on target server 
148              sites-enabled    # corresponds to /etc/nginx/sites-enable directory 
149                               # on target server 
150          srv                  # corresponds to /srv directory on the target server 
151            www                # corresponds to /srv/www directory on the target 
152                               # server 
153              50x.html         # corresponds to /srv/www/50x.html file on target 
154                               # server 
155              index.html       # corresponds to /srv/www/index.html file on target 
156                               # server
