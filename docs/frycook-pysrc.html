 28  ''' 
 29  Frycook depends on several things.  You'll see these things 
 30  passed to the classes and referenced from the classes. 
 31   
 32  settings dict 
 33  ============= 
 34   
 35  There are a few settings that frycook depends on.  They're 
 36  contained in a dictionary called settings that's passed 
 37  to the constructors for Cookbooks and Recipes.  It has the 
 38  following keys: 
 39   
 40  I{package_dir}: root of the packages hierarchy for the file 
 41  copying routines to look in 
 42   
 43  I{module_dir}: temporary directory for holding compiled mako 
 44  modules during template handling 
 45   
 46  I{file_ignores}: file patterns to ignore while copying package 
 47  files 
 48   
 49  environment dict 
 50  ================ 
 51   
 52  This contains all the metadata about the computers 
 53  the recipes and cookbooks will be applied to.  Most of its 
 54  data is directly relevant to specific recipes and is 
 55  filled in depending on the recipes' needs.  It's a  
 56  dictionary with three main sections that should always 
 57  be there: 
 58   
 59  I{users}: a list of the users recipes could reference, with such 
 60  things as public ssh keys and email addresses for them 
 61   
 62  I{computers}: a list of computers in the system, with such things 
 63  as ip addresses 
 64   
 65  I{groups}: groups of computers that will be addressed as a unit 
 66   
 67  example:: 
 68

 96  packages directory 
 97  ================== 
 98   
 99  This contains all the files needed by the recipes.  These are  
100  laid out the exact same as they will be on the target systems. 
101  Any files with a .tmplt extension will be processed as mako 
102  templates before being copied out to computers.  Here's an  
103  example layout:: 
104   
105      packages                # directory for the package files 
106        hosts                 # root for hosts package files 
107          etc                 # corresponds to /etc on the target server 
108            hosts.tmplt       # template that becomes /etc/hosts on the target server 
109        nginx                 # root for nginx package files 
110          etc                 # corresponds to /etc directory on the target server 
111            default           # corresponds to /etc/default directory on the target server 
112              nginx           # corresponds to /etc/default/nginx file on the target server 
113            nginx             # corresponds to /etc/nginx directory on target server 
114              nginx.conf      # corresponds to /etc/nginx/nginx.conf file on target server 
115              conf.d          # corresponds to /etc/nginx/conf.d directory on target server 
116              sites-available # corresponds to /etc/nginx/sites-available directory on target server 
117                default       # corresponds to /etc/nginx/sites-available/default directory on target server 
118              sites-enabled   # corresponds to /etc/nginx/sites-enable directory on target server 
119          srv                 # corresponds to /srv directory on the target server 
120            www               # corresponds to /srv/www directory on the target server 
121              50x.html        # corresponds to /srv/www/50x.html file on target server 
122              index.html      # corresponds to /srv/www/index.html file on target server
