 28  ''' 
 29  Recipes define subsystems that are distinct parts of larger systems.  They are 
 30  the basic units of setup in frycook.  Generally a recipe corresponds to a 
 31  package that needs to be installed or configured. 
 32   
 33  idempotence 
 34  =========== 
 35   
 36  One thing to keep in mind when creating recipes and cookbooks is idempotency. 
 37  By keeping idempotency in mind in general you can create recipes that you can 
 38  run again and again to push out minor changes to a package.  This way your 
 39  recipes become the only way that you modify your servers and can be a single 
 40  chokepoint that you can monitor to make sure things happen properly. 
 41   
 42  Lots of the cuisine functions you'll use have an "ensure" version that first 
 43  checks to see if a condition is true before applying it, such as checking if a 
 44  package is installed before trying to install it.  This is nice when things 
 45  could cause undesired configuration changes or expensive operations that you 
 46  don't want to happen every time.  These functions are a huge aid in writing 
 47  idempotent recipes and cookbooks. 
 48   
 49  rudeness 
 50  ======== 
 51   
 52  Another thing to keep in mind is that some actions performed in recipes can 
 53  affect the end users of the systems, in effect being rude to them.  This might 
 54  cause an outage or otherwise mess them up.  The recipe class keeps track of 
 55  whether or not this is ok in its 'ok_to_be_rude' variable so you can know what 
 56  actions are acceptable.  Consult this before doing rude things. 
 57   
 58  file set copying 
 59  ================ 
 60   
 61  The Recipe class defines a few helper functions for handling templates and 
 62  copying files to servers.  It runs files with a .tmplt extension through Mako, 
 63  using the dictionary you pass to it.  Regular files just get copied.  You can 
 64  specify owner, group, and permissions on a per-directory and per-file basis. 
 65   
 66  git repo checkouts 
 67  ================== 
 68   
 69  The Recipe class also defines some helper functions for working with git repos. 
 70  You can checkout a git repo onto the remote machine, or check it out locally 
 71  and copy it to the remote machine if you don't want to setup the remote machine 
 72  to be able to do checkouts. 
 73   
 74  apply process 
 75  ============= 
 76   
 77  This is where you apply a recipe to a server.  There are two class methods that 
 78  get called during the apply process, and possibly two messages that get 
 79  displayed.  Generally you'll just override the apply method and sometimes add 
 80  pre_apply or post_apply messages.  If you override pre_apply_checks, remember 
 81  to call the base class method.  Here's the order that things happen in: 
 82   
 83  pre_apply_message -> pre_apply_checks() -> apply() -> post_apply_message
