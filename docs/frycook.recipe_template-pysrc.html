 28  ''' 
 29  Recipes are the basic units of setup in frycook.  Generally a recipe 
 30  corresponds to a package that needs to be installed or a subsystem that needs 
 31  to be configured. 
 32   
 33  idempotence 
 34  =========== 
 35   
 36  One thing to keep in mind when creating recipes and cookbooks is idempotency. 
 37  By keeping idempotency in mind in general you can create recipes that you can 
 38  run again and again to push out minor changes to a package.  This way your 
 39  recipes become the only way that you modify your servers and can be a single 
 40  chokepoint that you can monitor to make sure things happen properly. 
 41   
 42  Lots of the cuisine functions you'll use have an "ensure" version that first 
 43  checks to see if a condition is true before applying it, such as checking if a 
 44  package is installed before trying to install it.  This is nice when things 
 45  could cause undesired configuration changes or expensive operations that you 
 46  don't want to happen every time.  These functions are a huge aid in writing 
 47  idempotent recipes and cookbooks. 
 48   
 49   
 50  apply 
 51  ===== 
 52   
 53  This is where you apply a recipe to a server.  There are two class methods that 
 54  get called during the apply process.  Generally you'll just override the apply 
 55  method.  If you override pre_apply_checks, remember to call the base class 
 56  method.  Here's the order that functions get called: 
 57   
 58  pre_apply_checks -> apply 
 59   
 60  cleanup 
 61  ======= 
 62   
 63  This is where you cleanup old recipe configurations from a server.  An example 
 64  is when I changed the home directory for my web server.  I first wrote a 
 65  cleanup that cleaned-up the old configuration, then an apply to apply the new 
 66  configuration.  That way you can always run the apply in the future when 
 67  building new machines and don't need the cleanup logic since the new machine 
 68  never had the old configuration that had to get cleaned-up. 
 69   
 70  file set copying 
 71  ================ 
 72   
 73  The Recipe class defines a few helper functions for handling templates and 
 74  copying files to servers.  It runs files with a .tmplt extension through make 
 75  using the dictionary you pass to it.  Regular files just get copied. 
 76  '''
